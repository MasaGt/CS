### 事象

- GraphQLスキーマを外部化したところ、 `node index.js` でエラーが発生するようになってしまった

    <img src="../img/Issue-Loading-Schema_1.svg" />

<br>

- 以下は `npm run start` 時のエラーログ
    <img src="../img/Issue-Loading-Schema_2.svg" />

---

### 原因

- pacakge.json に記述したビルドコマンドで、スキーマのコピーコマンド (src/schema をコピーするコマンド) が間違っていた

    - ★tsc では  .gql ファイルはコンパイル対象にならず、outDir に出力されないため、tsc コンパイル後に**自分で .gql ファイルを tsconfig の outDir で指定したディレクトリに配置する必要がある**

        <img src="../img/Issue-Loading-Schema_3.svg" />

---

### 解決策

- pakcage.json を意図した位置 (コンパイル後の index.js から参照できる位置) に配置するようにコピーコマンドを修正

    ```diff
    {
        "scripts": {
    -       "build": "tsc && cp -r src/schema dist/schema",
    +       "build": "tsc && cp -r src/schema dist/src/schema",
            "start": "npm run build && node dits/src/index.js",
            "codegen": "graphql-codegen"
        }
    }
    ```
